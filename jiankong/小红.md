## 第一步：确定 token

```bash
cat ~/.jiankong_token 2>/dev/null
```

**有值** → 记为 `TOKEN`，跳到第二步。

**无值**：
- 用户消息中包含 token（如"我的 token 是 xxx"）→ 执行初始化流程：
  1. 验证 token：
     ```bash
     curl -s -o /dev/null -w "%{http_code}" https://www.maxsou.cn/xiaohong/jiankong/user_stocks/<token>
     ```
     - 返回 `401` → 回复："该 token 未授权，请联系管理员分配有效 token"，停止
     - 返回 `200` → 保存：
       ```bash
       echo "<token>" > ~/.jiankong_token && echo "ok"
       ```
     - 回复："✅ token 已配置完成，后续直接使用监控功能无需再提供 token。"，停止
- 用户消息中不含 token → 回复："您还没有配置 token，请告诉我：小红，我的 token 是 xxx"，停止

### 【增加 / 修改 股票】

关键词：增加、添加、新增、修改、更新、调整成本、调整止盈、调整止损

**确定股票代码：**

- 用户输入已带后缀代码（如 `603107.SH`）→ 直接使用
- 用户输入纯数字代码（如 `603107`）→ 自动补全：6 开头 → `.SH`，0 或 3 开头 → `.SZ`
- 用户输入股票名称或关键词 → 查询：
  ```bash
  curl -s "https://www.maxsou.cn/xiaohong/jiankong/search_stock?q=<关键词>"
  ```
  - **返回空** → 回复"未找到「xxx」，请确认名称或代码"，停止
  - **唯一结果且名称完全一致** → 直接使用
  - **唯一结果但名称不完全一致** → 询问：`您是指「{name}（{ts_code}）」吗？(是/否)`
    - 是 → 使用；否 → 提示重新输入，停止
  - **多个结果** → 逐一展示询问：`1. {name}（{ts_code}）— 是/否？`
    - 用户回复"是"即使用，不再询问其余；全否则停止

**写入监控：**

```bash
curl -s -X POST https://www.maxsou.cn/xiaohong/jiankong/user_stocks/upsert \
  -H "Content-Type: application/json" \
  -d '{"token":"<TOKEN>","stock":{"ts_code":"<ts_code>","cost_price":<cost_price>,"profit_ratio":<profit_ratio>,"loss_ratio":<loss_ratio>}}'
```

### 【开启定时提醒】

关键词：每X分钟、每X小时、定时提醒、自动提醒、开启提醒

**1. 读取通知时间窗口配置：**

```bash
cat ~/.jiankong_schedule 2>/dev/null
```

若文件不存在，使用默认值：
```json
{"days": "1-5", "start": "09:25", "end": "15:00"}
```

**2. 提取提醒间隔：**
- 每 N 分钟 → cron: `*/N * * * <days>`
- 每 N 小时 → cron: `0 */N * * <days>`
- 未指定 → 默认 10 分钟

其中 `<days>` 取自配置的 `days` 字段（如 `1-5` 表示周一到周五）。

**3. 安装监控脚本（首次或更新时执行）：**

找到 `小红.md` 所在目录，将 `scripts/monitor.sh` 复制到用户目录：

```bash
SKILL_DIR="$(dirname $(realpath ~/.claude/commands/小红.md))"
cp "$SKILL_DIR/scripts/monitor.sh" ~/.jiankong_monitor.sh
chmod +x ~/.jiankong_monitor.sh
```

**4. 移除旧 crontab 条目（如有）：**

```bash
crontab -l 2>/dev/null | grep -v jiankong_monitor | crontab -
```

**5. 注册新 crontab：**

```bash
(crontab -l 2>/dev/null; echo "*/<N> * * * * ~/.jiankong_monitor.sh") | crontab -
```

将 `<N>` 替换为实际间隔分钟数。

**6. 回复用户：**

> ✅ 已开启定时提醒，每 X 分钟检查一次（{days_label} {start}～{end}）。
> 报告写入 `~/.jiankong_monitor.log`。
> 说「小红，停止提醒」可取消，「小红，修改通知时间」可调整时间段。

### 【修改通知时间】

关键词：修改时间、调整时间、通知时间、几点到几点、工作日、周末

从用户输入中提取：
- **days**：周一到周五 → `1-5`；每天 → `*`；周一到周六 → `1-6`；用户指定具体日期则对应转换
- **start**：开始时间，格式 `HH:MM`
- **end**：结束时间，格式 `HH:MM`
- 未提及的字段保留现有配置

保存配置：

```bash
echo '{"days":"<days>","start":"<start>","end":"<end>"}' > ~/.jiankong_schedule
```

若当前有正在运行的定时任务（`~/.jiankong_cron_id` 有值），询问用户："是否立即用新时间段重启定时提醒？(是/否)"
- 是 → 按【开启定时提醒】流程重新创建任务
- 否 → 仅保存配置，下次开启时生效

回复："✅ 通知时间已更新：{days_label} {start}～{end}"

---

### 【停止定时提醒】

关键词：停止提醒、取消提醒、关闭提醒、不用提醒了

1. 检查是否有定时任务：
   ```bash
   crontab -l 2>/dev/null | grep jiankong_monitor
   ```
   - 无输出 → 回复："当前没有运行中的定时提醒"，停止

2. 移除 crontab 条目：
   ```bash
   crontab -l 2>/dev/null | grep -v jiankong_monitor | crontab -
   ```

3. 回复用户："✅ 定时提醒已停止。"
